## <a name="introduction"></a><span data-ttu-id="17459-104">介绍</span><span class="sxs-lookup"><span data-stu-id="17459-104">Introduction</span></span>

<span data-ttu-id="17459-105">预测性维护 (PdM) 可预测维护需求，以避免计划外的停机产生成本。</span><span class="sxs-lookup"><span data-stu-id="17459-105">Predictive Maintenance (PdM) anticipates maintenance needs to avoid costs associated with unscheduled downtime.</span></span> <span data-ttu-id="17459-106">通过连接到设备和监视设备生成的数据，可以标识导致发生潜在问题或故障的模式。</span><span class="sxs-lookup"><span data-stu-id="17459-106">By connecting to devices and monitoring the data the devices produce, you can identify patterns that lead to potential problems or failures.</span></span> <span data-ttu-id="17459-107">然后，在这些问题发生之前，可以使用这些见解来解决问题。</span><span class="sxs-lookup"><span data-stu-id="17459-107">You can then use these insights to address issues before they happen.</span></span> <span data-ttu-id="17459-108">这一预测设备或资产何时需要进行维护的功能，使用户能够优化设备生存期并最大限度地减少停机时间。</span><span class="sxs-lookup"><span data-stu-id="17459-108">This ability to predict when equipment or assets need maintenance allows you to optimize equipment lifetime and minimize downtime.</span></span>

<span data-ttu-id="17459-109">PdM 从车间的设备所生成的数据中提取见解，并依照这些见解执行操作。</span><span class="sxs-lookup"><span data-stu-id="17459-109">PdM extracts insights from the data produced by the equipment on the shop floor and acts on these insights.</span></span> <span data-ttu-id="17459-110">Pdm 的概念可追溯到 20 世纪 90 年代初，并扩展了常规计划的预防性维护。</span><span class="sxs-lookup"><span data-stu-id="17459-110">The idea of PdM goes back to the early 1990&#39;s and augments regularly scheduled, preventive maintenance.</span></span> <span data-ttu-id="17459-111">在早期，由于缺乏生成数据的可用传感器以及缺乏收集数据和分析数据的计算资源，所以很难实现 Pdm。</span><span class="sxs-lookup"><span data-stu-id="17459-111">Early on, the lack of availability of sensors generating data as well as a lack of computational resources for gathering and analyzing the data made it difficult to implement PdM.</span></span> <span data-ttu-id="17459-112">今天，物联网 (IoT)、云计算、数据分析和机器学习 (ML) 的发展使预测性维护能够走向主流。</span><span class="sxs-lookup"><span data-stu-id="17459-112">Today, advances in the Internet of Things (IoT), cloud computing, data analytics, and Machine Learning (ML) are enabling predictive maintenance to go mainstream.</span></span>

<span data-ttu-id="17459-116">为本文提供一些背景知识后，我们将探讨如何结合使用本地数据、Azure 机器学习来实现各个部分的 Pdm 解决方案以及机器学习模型的使用。</span><span class="sxs-lookup"><span data-stu-id="17459-116">After introducing some background in this article, we discuss how to implement the various pieces of a PdM solution using a combination of on-premises data, Azure machine learning, and usage of the machine learning models.</span></span> <span data-ttu-id="17459-117">Pdm 在进行决策时非常依赖于数据，因此，我们先来了解数据收集。</span><span class="sxs-lookup"><span data-stu-id="17459-117">PdM relies heavily on data to make decisions, so we start by looking at data collection.</span></span> <span data-ttu-id="17459-118">必须收集数据，然后将其用于评估现在发生的情况，并用于在以后生成预测性更好的模型。</span><span class="sxs-lookup"><span data-stu-id="17459-118">The data must be collected and then used to evaluate what is happening now as well as used to build up better predictive models in the future.</span></span> <span data-ttu-id="17459-119">最后，我们将解释分析解决方案是什么，其中包括在报表工具（如 [Power BI](https://docs.microsoft.com/power-bi/)）中可视化分析结果。</span><span class="sxs-lookup"><span data-stu-id="17459-119">Finally, we explain what an analysis solution looks like, including visualizing the analysis results in a reporting tool like [Power BI](https://docs.microsoft.com/power-bi/).</span></span>

## <a name="maintenance-strategies"></a><span data-ttu-id="17459-120">维护策略</span><span class="sxs-lookup"><span data-stu-id="17459-120">Maintenance Strategies</span></span>

<span data-ttu-id="17459-121">在制造业的历史上，出现了几种维护策略。</span><span class="sxs-lookup"><span data-stu-id="17459-121">Over the history of the manufacturing industry, several maintenance strategies emerged.</span></span> <span data-ttu-id="17459-122">响应式维护在问题出现后修复问题。</span><span class="sxs-lookup"><span data-stu-id="17459-122">Reactive maintenance fixes issues after they occur.</span></span> <span data-ttu-id="17459-123">预防性维护按照基于以前发生的故障的维护计划在问题出现前修复问题。</span><span class="sxs-lookup"><span data-stu-id="17459-123">Preventive maintenance fixes issues before they occur by following a maintenance schedule based on prior failure experience.</span></span> <span data-ttu-id="17459-124">Pdm 也是在问题出现前修复问题，但会考虑设备的实际使用情况，而不是使用固定的计划。</span><span class="sxs-lookup"><span data-stu-id="17459-124">PdM also fixes issues before they occur but considers the actual utilization of the equipment instead of working from a fixed schedule.</span></span> <span data-ttu-id="17459-125">在以上三种维护策略中，Pdm 是最难实现的维护策略，这是因为过去在数据收集、处理和数据可视化方面存在限制。</span><span class="sxs-lookup"><span data-stu-id="17459-125">Of the three, PdM was the most difficult to achieve because of historical limitations on data collection, processing, and data visualization.</span></span> <span data-ttu-id="17459-126">让我们更为深入地了解每个选项。</span><span class="sxs-lookup"><span data-stu-id="17459-126">Let&#39;s look at each option in a bit more detail.</span></span>

<span data-ttu-id="17459-127">响应式维护体现的是&quot;如果没有出现故障，则不对其进行修复&quot;的思维模式。</span><span class="sxs-lookup"><span data-stu-id="17459-127">Reactive maintenance embodies the &quot;if it isn&#39;t broken, don&#39;t fix it&quot; mentality.</span></span> <span data-ttu-id="17459-128">仅在资产出现问题时才对其进行维护。</span><span class="sxs-lookup"><span data-stu-id="17459-128">Service the asset only when it fails.</span></span> <span data-ttu-id="17459-129">例如，只有在你的 5 轴 CNC 加工中心的电机停止工作时才对其进行维护。</span><span class="sxs-lookup"><span data-stu-id="17459-129">For example, the motor of your 5-axis CNC Machining Center gets serviced only when it stops working.</span></span> <span data-ttu-id="17459-130">响应式维护最大程度地延长最终出现故障的组件的生存期。</span><span class="sxs-lookup"><span data-stu-id="17459-130">Reactive maintenance maximizes the lifetime of the component which eventually fails.</span></span> <span data-ttu-id="17459-131">响应式维护还导致未知的停机时间、由发生故障的组件导致的对组件的意外附加伤害，以及其他问题。</span><span class="sxs-lookup"><span data-stu-id="17459-131">Reactive maintenance also introduces unknown amounts of downtime, unexpected collateral damage to components harmed by the failing component, and other issues.</span></span>

<span data-ttu-id="17459-132">预防性维护要求你在预先确定的时间间隔维护资产。</span><span class="sxs-lookup"><span data-stu-id="17459-132">Preventive maintenance requires you to the service the asset at pre-determined intervals.</span></span> <span data-ttu-id="17459-133">时间间隔通常基于资产在过去曾出现故障的频率。</span><span class="sxs-lookup"><span data-stu-id="17459-133">The interval is typically based on the experienced failure frequency history for the asset.</span></span> <span data-ttu-id="17459-134">这些时间间隔基于历史记录性能、模拟、统计建模等。此策略的优点是，它增加了正常运行时间、减少了故障次数并可计划维护。</span><span class="sxs-lookup"><span data-stu-id="17459-134">These intervals are based on historical performance, simulations, statistical modeling, etc. The advantage of this strategy is that it increases uptime, results in less failures, and maintenance can be planned.</span></span> <span data-ttu-id="17459-135">在许多情况下，其缺点是资产上被替换的组件可能还具有一些生存期。</span><span class="sxs-lookup"><span data-stu-id="17459-135">The downside in many cases is the replaced component on the asset may have had some life left.</span></span> <span data-ttu-id="17459-136">此结果导致过度维护和浪费。</span><span class="sxs-lookup"><span data-stu-id="17459-136">This results in over-maintenance and waste.</span></span> <span data-ttu-id="17459-137">另一方面，某些部分在计划的维护到来前仍可能出现故障。</span><span class="sxs-lookup"><span data-stu-id="17459-137">On the flip side, some parts may still fail before the scheduled maintenance.</span></span> <span data-ttu-id="17459-138">你可能很了解预防性维护：在固定的若干小时的操作（或某些其他指标）后，停止使用机器并检查机器。</span><span class="sxs-lookup"><span data-stu-id="17459-138">You probably know preventive maintenance well: after every set hours of operation (or some other metric), stop using the machine and inspect the machine.</span></span> <span data-ttu-id="17459-139">替换任何在到期时间应被替换的部件。</span><span class="sxs-lookup"><span data-stu-id="17459-139">Replace any parts which are due to be replaced.</span></span>

<span data-ttu-id="17459-140">预防性维护使用模型来预测资产可能出现组件故障的时间，以监视资产的使用情况。</span><span class="sxs-lookup"><span data-stu-id="17459-140">Predictive maintenance monitors the usage of assets using models to predict when an asset will likely experience a component failure.</span></span> <span data-ttu-id="17459-141">然后，组件将具有针对实时维护&quot;&quot;计划的维护。</span><span class="sxs-lookup"><span data-stu-id="17459-141">That component then has its maintenance scheduled for &quot;just-in-time maintenance&quot;.</span></span> <span data-ttu-id="17459-142">PdM 通过最大化正常运行时间和资产生命周期在以前的策略之上进行了改进。</span><span class="sxs-lookup"><span data-stu-id="17459-142">PdM improves on the previous strategies by maximining both uptime and asset life.</span></span> <span data-ttu-id="17459-143">因为你在接近组件最大生存期的时候维护设备，所以替换工作组件的成本会更低。</span><span class="sxs-lookup"><span data-stu-id="17459-143">Since you service the equipment close to the component maximum lifetimes, you spend less money on replacing working parts.</span></span> <span data-ttu-id="17459-144">缺点是 PdM 的实时特性更难执行，因为它要求具有更好的响应性和更为灵活的服务组织。</span><span class="sxs-lookup"><span data-stu-id="17459-144">The downside is that the just-in-time nature of PdM is more difficult to execute since it requires a more responsive and flexible Services organization.</span></span> <span data-ttu-id="17459-145">回到 5 轴 CNC 加工中心的电机的例子，例如，如果预测模型预测电机在未来 24 小时内出现故障的可能性的概率为 75%（基于来自机器中的传感器的信息），则应&quot;合理&quot;地计划其维护（例如，在不中断生产的情况下预先计划）。</span><span class="sxs-lookup"><span data-stu-id="17459-145">Back to the motor of the 5-axis CNC Machining, you would schedule its maintenance &quot;gracefully&quot; (i.e., in a planned fashion, without disrupting production) if a predictive model predicts that the motor has, say, a 75% probability of failure in the next 24 hours (based on the information coming from sensors in the machine).</span></span>
